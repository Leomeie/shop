## Codebase Patterns (cont.)
- When testing user isolation, each test client needs its own `APIClient()` instance — sharing `api_client` across fixtures causes auth state collisions
- Cart prices are in fen (cents); views divide by 100 for yuan display — test assertions must use matching math
- `CartItem` has `unique_together = ("user", "sku")` — `CartService.add()` uses `get_or_create` to handle this
- DRF throttle rates are class-level (`SimpleRateThrottle.THROTTLE_RATES`) evaluated at import time — `override_settings` doesn't affect them; patch the class attribute directly
- Custom exception handler wraps all responses in `{"code", "message", "errors"}` — test 429/400/404 assertions must check `resp.data["message"]` not `resp.data["detail"]`
- Frontend API errors: `request.js` interceptor shows ElMessage for 401/403/404/500/network; 400 validation errors are NOT shown (views handle field-level errors via `error.errors`)
